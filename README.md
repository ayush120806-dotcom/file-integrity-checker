A File Integrity Monitor (FIM) is a security tool designed to detect unauthorized or unexpected changes in files and directories. The primary objective of a FIM is to ensure the integrity, authenticity, and reliability of critical system files by continuously monitoring them and alerting users whenever modifications occur. This is especially important in modern computing environments where data breaches, malware infections, and insider threats can compromise sensitive information.The File Integrity Monitor developed in this project is a Python-based solution that uses cryptographic hashing techniques to track changes in files. It leverages the built-in hashlib library to generate secure hash values (specifically SHA-256) for each file in a selected directory. A hash value is a unique digital fingerprint of a file’s contents. Even a minor change in the file, such as modifying a single character, results in a completely different hash value. This property makes hashing an effective method for detecting file tampering.When the tool is executed for the first time, it scans all files within the specified directory and calculates their hash values. These values are then stored in a local database file (in JSON format). This initial scan serves as a baseline snapshot of the system’s state. During subsequent executions, the tool recalculates the hash values of the current files and compares them with the previously stored values. Based on this comparison, it identifies three types of changes: modified files (existing files whose content has changed), new files (files that were not present during the previous scan), and deleted files (files that were present earlier but are now missing).The tool provides a clear and structured report indicating all detected changes, along with timestamps for when the scan was performed. This enables users or system administrators to quickly identify suspicious activities and take appropriate action. For example, if a critical configuration file is unexpectedly modified, it could indicate a potential security breach or malware activity.One of the key advantages of this File Integrity Monitor is its simplicity and portability. Since it is implemented in Python and uses standard libraries, it does not require any complex setup or external dependencies. It can be easily deployed on different systems and customized based on user requirements, such as monitoring specific file types or directories.However, it is important to note that this tool performs periodic scanning rather than real-time monitoring. This means changes are detected only when the tool is executed. Future enhancements could include real-time monitoring using file system event listeners, email alerts, integration with logging systems, or a graphical user interface for improved usability.In conclusion, the File Integrity Monitor is a practical and efficient tool for maintaining data integrity and enhancing system security. By providing visibility into file changes, it helps users detect potential threats early and maintain trust in their systems.
